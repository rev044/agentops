# Release v2.21.0

## Highlights

Codex is now treated as a first-class runtime instead of a compatibility
layer. This release tightens the generated skill bundle, adds stronger
guardrails around next-work carry-forward in RPI, and hardens the local
validation stack so release and pre-push checks fail for the right reasons.
It also fixes a streamed-phase timeout regression that could leave phase runs
and package tests hanging longer than intended.

## What's New

- Codex-first skills now ship with explicit operator contracts, override
  coverage checks, and generated-bundle parity validation.
- The next-work queue now tracks claim-aware, per-item lifecycle state with
  stronger contract enforcement across `/rpi` workflows.
- Headless runtime skill smoke tests and backbone prompt validation now run as
  part of local release validation.
- Git/worktree detection is more reliable in hooked and linked-worktree
  environments.
- Streamed phase timeouts now unwind promptly instead of hanging on canceled
  stdout reads.

## All Changes

### Added

- A Codex-first maintenance model with override contracts, parity checks, and
  generated-artifact governance.
- Claim-aware next-work lifecycle enforcement for follow-on RPI work.
- Headless runtime smoke coverage and Codex backbone prompt validation in the
  release gate.

### Changed

- Codex maintenance docs and CLI-to-skill mapping to better reflect the
  shipped runtime model.
- Release-prep validation and runtime smoke expectations to reduce false
  failures and stale generated artifacts.

### Fixed

- Next-work queue races during concurrent updates.
- Prompt drift between Codex overrides and generated artifacts.
- Git and worktree context handling in vibe-check and related runtime paths.
- Nested pre-push validation wrappers and other push-gate regressions.
- Streamed runtime phase timeout cancellation.
