## Highlights

This release makes Codex native hooks the default AgentOps path on Codex CLI
v0.115.0+. The install flow now lands as a real native plugin with
`~/.codex/hooks.json`, so Codex startup and closeout behave like a first-class
runtime instead of a fallback story. This is the release that materially reduces
Codex install friction.

## What's New

- **Codex native hooks by default** — `scripts/install-codex-plugin.sh` and the
  public `scripts/install-codex.sh` path now center the native plugin and native
  hook manifest under `~/.codex/`.
- **Same lifecycle surface as Claude** — Codex now gets native SessionStart,
  SessionEnd/Stop-equivalent automation, and the same operational hook contract
  where the runtime supports it.
- **Cleaner migration path** — install now archives stale raw mirrors in
  `~/.agents/skills` and `~/.codex/skills` when they overlap with the plugin
  cache, which reduces duplicate-skill drift and onboarding confusion.
- **Knowledge compiler rename** — athena is now `/compile`, with Karpathy-style
  incremental knowledge compilation into `.agents/compiled/`.
- **CLI hardening for future work** — more `cmd/ao` logic moved behind injected
  seams and internal packages, which makes the release surface easier to test and
  evolve.
