# v2.23.0 Release Notes

## Highlights

The RPI lifecycle is now decomposed into independently invocable phases.
`/discovery` handles research, planning, and pre-mortem in one pass;
`/validation` handles vibe-check and post-mortem. You can still use `/rpi`
for the full loop — it delegates to these under the hood.

The pre-push gate gained a `--fast` mode that only validates packages
touched by your diff, cutting gate time significantly on focused changes.

Test quality improved substantially: 271 new test functions, four internal
packages at 100% coverage, and ~50 former smoke tests upgraded from
zero-assertion to real behavioral checks.

## What's New

- **Phase orchestrators** — `/discovery` and `/validation` are first-class
  skills, composable independently or via `/rpi`
- **Stigmergic packet scorecard** — Knowledge flywheel now ranks packets by
  utility so high-value learnings surface first
- **Pinned work queue in /evolve** — Direct the improvement loop at specific
  targets with blocker auto-resolution
- **Per-package coverage ratchet** — Baselines that only move up, enforced
  on every push
- **Fast pre-push** — `scripts/pre-push-gate.sh --fast` checks only what
  changed
- **Standards in workers** — Go/Python coding standards auto-injected into
  `/crank` and `/swarm` parallel workers

## Bug Fixes

- Goal runner deadlock and missing job timeouts
- 17 CLI correctness issues from deep audit (leaks, races, panics, nil checks)
- Session close reliability improvements
- Test file naming hygiene (`_extra_test.go` → canonical names)
- Auto-extract quality gate to filter low-fidelity learnings
