# AgentOps v2.24.0

Six skills enhanced with battle-tested patterns from gstack validation. Pre-mortem, vibe, post-mortem, pr-prep, council, and standards all gain new capabilities that compound through the knowledge flywheel.

## Highlights

- **Predict-then-verify loop** — Pre-mortem predictions now carry unique IDs through vibe correlation and post-mortem scoring (HIT/MISS/SURPRISE), closing the feedback loop on plan quality.
- **Domain-aware validation** — Vibe auto-loads SQL safety, LLM trust boundary, and race condition checklists from the standards skill when domain signals are detected.
- **Signal-to-noise filtering** — Vibe findings classified as CRITICAL vs INFORMATIONAL, with project-level suppression patterns for known false positives.

## What's New

### Pre-mortem
- **Error & rescue map** (Step 2.5) — Tabular risk/mitigation format with 3 worked examples
- **Scope mode selection** (Step 1.6) — Expand/Hold/Reduce framework with auto-detection from goal text
- **Temporal interrogation** (Step 2.4) — Hour 1/2/4/6+ timeline walkthrough for time-dependent risks
- **Prediction tracking** — Findings get `pm-YYYYMMDD-NNN` IDs for downstream correlation

### Vibe
- **Finding classification** — CRITICAL (blocks ship) vs INFORMATIONAL separation
- **Suppression framework** — Default + project-level patterns for known false positives
- **Domain-specific checklists** — Auto-loaded from standards when SQL/LLM/concurrency detected
- **Pre-mortem prediction correlation** (Step 3.6) — Maps findings back to prediction IDs

### Post-mortem
- **RPI session streak tracking** (Step 1.5) — Consecutive session days and verdict history
- **Prediction accuracy scoring** (Step 3.5) — HIT/MISS/SURPRISE against pre-mortem predictions
- **Persistent retro history** (Step 4.8) — Structured JSON to `.agents/retro/` for trend analysis

### PR-prep
- **Commit split advisor** (Phase 4.5) — Suggests bisectable commit ordering (suggestion-only)

### Council
- **Finding auto-extraction** — WARN/FAIL findings staged for flywheel consumption

### Standards
- SQL safety checklist, LLM trust boundary checklist, race condition checklist
