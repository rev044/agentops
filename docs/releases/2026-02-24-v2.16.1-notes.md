# v2.16.1

## Highlights

- **`--json` flag standardized** ‚Äî All `ao` commands now accept `--json` as a shorthand for `-o json`, matching the `bd` CLI convention. Consistency across 25 Go files, 7 hooks, and skill documentation.
- **Security gate now blocks CI** ‚Äî Security findings (gosec, gitleaks, semgrep, trivy, hadolint) are now blocking. Quality findings (golangci-lint, radon, ruff) remain non-blocking warnings.
- **`ao rpi parallel`** ‚Äî New command for parallel task execution in RPI workflows.

## What's New

### `--json` Flag Consistency
Community contributor @tim-coutinho identified that `bd` uses `--json` while `ao` used `-o json`. This release standardizes both CLIs: `--json` is now a persistent root flag that sets `-o json` under the hood. All documentation, hooks, and examples updated.

### Security Gate Improvements
The CI security toolchain gate was always failing because quality findings (linter warnings) were mixed with security findings (actual vulnerabilities). The gate now separates the two: security findings block merges, quality findings produce warnings. Stale `.claude/worktrees/` directories are excluded from scans to prevent inflated finding counts.
