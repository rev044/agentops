# Release v2.22.0

## Highlights

Council findings are now saved to a persistent registry so agents stop
repeating the same mistakes. This release also adds per-repo execution
profiles, a headless multi-agent backend for CI, fixes 23 CLI bugs, and
hardens post-mortem evidence capture.

## What's New

- **Finding registry** — Council findings are written to a durable
  registry (`.agents/findings/registry.jsonl`) and automatically fed back
  into planning and validation, so the same class of bug is caught
  earlier next time instead of rediscovered.
- **Repo execution profiles** — A `.repo-execution-profile.json` file
  lets skills and runtimes adapt to each repository's startup reads,
  validation gates, and done-criteria instead of running generic prompts.
- **Headless team backend** — Multi-agent workflows can now run
  non-interactively (e.g. in CI) with `--dangerously-skip-permissions`,
  `--max-turns`, and structured JSON output.
- **23 CLI bug fixes** — A full audit fixed 7 high-severity issues
  (goroutine leaks, race conditions, panics), 11 medium (buffer
  overflows, missing error handling), and 5 low (nil checks, field
  inconsistencies).
- **Post-mortem evidence hardening** — Staged changes and worktree
  evidence are now captured durably so proof isn't lost when sessions
  compact or repos are cleaned up.

## All Changes

### Added

- Persistent finding registry with council write/read-path support.
- Per-repo execution profiles with skill and runtime wiring.
- Headless multi-agent backend for CI and automated workflows.

### Changed

- Synced Codex bundles, embedded standards, and install artifacts after
  branch merge.
- Recorded validation-cycle feedback into tracked learnings.

### Fixed

- `ao lookup` and inject scoring for findings after branch merge.
- 23 CLI bugs (goroutine leaks, race conditions, panics, buffer
  overflows, missing error handling, nil checks).
- Post-mortem evidence capture across compaction and cleanup.
