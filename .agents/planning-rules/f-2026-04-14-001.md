# Planning Rule: Production command refactors can miss the paired test diff expected by the command/test-pairing gate.

Prevent this known failure mode:

- Pattern: Production command refactors can miss the paired test diff expected by the command/test-pairing gate.
- Ask: When a slice changes a production file under cli/cmd/ao, does the same slice add or adjust direct command tests that satisfy the pairing gate?
- Do: Plan the production-code helper split and the matching command test diff together, then run scripts/pre-push-gate.sh --fast before push.
